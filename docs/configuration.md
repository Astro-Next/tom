# Configuration

## Config file

Tom's configuration lives in `.tom/settings.json` in the project root. Created by `tom init`.

## Schema

```json
{
  "id": "string",
  "patrol": {
    "interval": "30m"
  },
  "retro": {
    "interval": "1d",
    "time": "22:00"
  },
  "agent": {
    "timeout": "30m",
    "maxRetries": 2
  },
  "dev": {
    "concurrent": 2
  },
  "review": {
    "concurrent": 2
  }
}
```

## Field reference

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | string | auto-generated | Unique project identifier. Generated by `tom init`. Used for runtime directory paths. |
| `patrol.interval` | string | `"30m"` | How often patrol runs. Format: digits + `m` (minutes) or `h` (hours). Examples: `30m`, `1h`. |
| `retro.interval` | string | `"1d"` | How often retro runs. Format: digits + `d` (days). Examples: `1d`, `2d`. |
| `retro.time` | string | `"22:00"` | What time retro runs. Format: `HH:MM` in 24-hour time. |
| `agent.timeout` | string | `"30m"` | Maximum wall time for any Claude Code subprocess before it is terminated. Format: digits + `m` or `h`. Applies to all agent types. |
| `agent.maxRetries` | number | `2` | Maximum failed attempts before escalating to `blocked` (dev/review) or giving up (pm/retro). Applies to all agent types. |
| `dev.concurrent` | number | `2` | Maximum dev agents running simultaneously. |
| `review.concurrent` | number | `2` | Maximum review agents running simultaneously. |

## Validation rules

| Field | Rule |
|-------|------|
| `patrol.interval` | Must match `^\d+[mh]$` (e.g. `30m`, `1h`) |
| `retro.interval` | Must match `^\d+d$` (e.g. `1d`, `2d`) |
| `retro.time` | Must be valid `HH:MM` in 24-hour format (00:00 to 23:59) |
| `agent.timeout` | Must match `^\d+[mh]$` (e.g. `30m`, `2h`) |
| `agent.maxRetries` | Positive integer |
| `dev.concurrent` | Positive integer |
| `review.concurrent` | Positive integer |

## Target project structure

Tom expects certain files in the repository it manages. These are created by `tom init` if they don't exist.

```
.tom/
  settings.json          # Tom configuration
.worktrees/              # Agent worktrees (gitignored)
CLAUDE.md                # Instructions for Claude Code agents
CONVENTIONS.md           # Coding patterns, naming, style rules
docs/
  index.md               # Project knowledge base entry point
```

Runtime files are stored outside the repo:
- `~/.tom/{project-id}/patrol.lock` — patrol process lock file (holds the running PID)
- `~/.tom/{project-id}/retro.lock` — retro process lock file (holds the running PID)
- `~/.tom/{project-id}/logs/tom.log` — runtime logs
- `/tmp/tom-{project-id}/cache/` — attachment cache

### CLAUDE.md

Instructions that all Claude Code agent subprocesses read before doing work. Contains project-specific rules, setup commands, and constraints. This is the standard Claude Code project file — agents inherit it automatically when running in the worktree.

### CONVENTIONS.md

Coding patterns, naming conventions, and project standards. Both dev and review agents read this before writing or reviewing code. Retro may propose updates to this file.

### docs/index.md

Entry point to the project knowledge base. Agents read this to understand the project's domain, architecture, and key decisions. Links to other docs as needed.
